The Disappearence of Dr. Viator

Originality

  • This game isn't original in the concept, it's a point and click game where you look for clickable items and it plays like a hidden objects game. I intended to place more originality through the game set up; I was hoping to get more rooms, more objects, and more story involved. I should be able to do that with the next iteration as I now have a better understanding on how to implement the game.

    • Technical Merit

  • Buttons and functions. The entire game is a group of buttons working with each other. I think the way I implemented the buttons is a bit tedious, I made a function for each one, one to open the clue and then another function to close. It wasn't until later that I realized that I could probably merge them into one function and pass through the game object. I will be working towards making it more intuitive, code-wise. To hide the objects, since I didn't want to destroy them, I moved to the side of the screen. So they still exist, but hidden. This is extremely inefficient but I couldn't find a better way to do it. So, currently, clicking an item just creates a new instance and I'm sure at some point, it would make the game overbearing as the images are just getting thrown to a pile.

    • Prototype Postmortem:

  • I learned the usage of buttons using .json files. It was a lot easier than I thought it would be. I was apprehensive in using the type because I was afraid of editing the file templates. Everything turned out well, I also figured out that I wouldn't have to make a file for each button. I made a "group" that used the same format and just made sure every button I create thereafter matched the file criteria. It worked well for the most part, but for bigger sized items I'll have to make a new template.

    • Prototype Goal:

  • Have clues that show something and have it hide when closed. I think my prototype did this, just not as well as it could have

    • Player Experience Goals:

  • Learning about new places. I've always been interested in using games as educational tools. I think the range we currently have for educational purposes feels a little gimmicky and so don't get played with fervor. I want to create games that are fun to play, nice to look at, and are engaging so that people keep coming back to play and learn a little more, or come back to play because it was fun.

    • Gameplay:

  • The player clicks the buttons around the room and tries to figure out the city/country based onn the clues shown (in a finished game, there would be a box to input guesses). The player can use the internet to their heart's content to search how the clues all relate to each other and in the process learn more about different countries or different cities. For this prototype, for example, the lilies on the table reference Italy's national flower.

    • Strategies:

  • Items will "light up" when hovered over to show it's clickable. Moving the mouse around the screen helps in finding clues.

    • Story/Setting/Premise:

  • Dr. Viator, the famous historian, has suddenly disappeared. No one knows where she could be! The media is speculating that she's been kidnapped. There are some that say she's messed with something big and put her nose in business she wasn't meant to know (she does that a lot). The police think she could have been murdered. As her new assistant, you are the first suspect. She mentioned taking a vacation soon, she probably forgot to tell you when that would be. Find Dr. Viator to clear your name. Where could she be?
